1. Introduction to PyDAO by Example

PyDAO is very thin object-relational mapper similar to Hibernate (but much simpler). It's created to speed-up application development. It's very simple, but powerful, based on POPO (Plain Old Python Objects).

1.1. Simple CRUD (Create/Read/Update/Delete) example

First of all, we have to define our object that we will operate on:

Listing 1. Simple POPO (Plain Old Python Object)
class TEST_USER:

    def __init__(self):
        self.id = None
        self.login = None
        self.salary = None
        self.password = None
        self.companyID = None

As you can see, no fields types have to be declared. Then we can insert first object into database:

Listing 2. Inserting of objects into database
user1 = TEST_USER()
user1.login = "user1"
user1.salary = 5000.0
self.dao.save(user1)

user2 = TEST_USER()
user2.login = "user2"
user2.salary = 7000.0
self.dao.save(user2)

Above code works with no change with InMemoryDao and MysqlDao implementations. This allows you to start with in memory database and continue if database scheme is stable.

Now object is inserted into database and is ready to be read:

Listing 3. Getting list of objects from database
userExample = TEST_USER()
userExample.login = "user1"
self.dao.list(userExample)

As you can see wee specify fields that should be used to filter data and PyDAO returns list of matching objects (one in this case). Similar technique is used to update existing records into database. We must specify id field and fields that should be changed:

Listing 4. Changing data of existing objects
userExample = TEST_USER()
userExample.id = user1.id
userExample.salary = 6000.0
self.dao.update(userExample)

At least we may want to delete some records from database:

Listing 5. Deleting of objects based on example
userExample = TEST_USER()
userExample.salary = InRangeCondition.InRangeCondition(0, 6500)
self.dao.delete(userExample)

Notice that we are using here more powerful technique to select subset of record (see: InRangeCondition.py).
